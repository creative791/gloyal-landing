## Жёсткие правила клиента (НЕ нарушать)
1. ⛔ **НИКОГДА не добавляй letter-spacing / tracking** — ни на заголовках, ни на тексте. Никакого кернинга.
2. Заголовки и все тайтлы — `font-display`. Весь прочий текст — `font-body`.
3. **НЕТ рамок/обводок (border, stroke) на карточках/плитках/пилюлях («плашках»)**. Обводки допустимы ТОЛЬКО на иконках.
4. Все иконки — тонкая линия 1px (стиль lucide), рисуй инлайн-SVG с `stroke-width="1"` (или 1.5 если в макете толще). НЕ скачивай иконки картинкой.
5. Плашки компактные, без лишней пустоты. Стекло — деликатное.
6. Надзаголовки (эйбрау над h2: «Кейсы», «Нам доверяют» и т.п.) — приглушённый/серый (`text-text-muted`).

## Конвенции компонента
- Файл: `src/sections/<Name>.tsx`. Default export — React-компонент `<Name>` без пропсов.
- Обёртка — семантический `<section>`. Контент по центру: десктоп контейнер `max-w-[1280px] mx-auto px-20` (Figma: 1440 ширина, поля 80px). Мобайл: `px-4`. Сверяй реальные отступы по Figma.
- **Адаптив**: база (без префикса) = МОБИЛЬНЫЙ макет (390px), `lg:` (1024px+) = ДЕСКТОП макет (1440px). Читай ОБА фрейма (desktop + mobile node) и реализуй один отзывчивый компонент.
- **Без анимаций** — только статика, чистая структура (анимации добавим позже отдельным этапом).

## Ассеты
- Растровые ассеты (3D-маскоты «G», волны/сквиглы, размытые blob-эллипсы, логотипы) — скачивай через MCP `download_assets`, сохраняй в `public/assets/<section>/<имя>.<ext>`, ссылайся как `/assets/<section>/...`.
- Логотипы брендов (интеграции/доверие) — предпочтительно SVG (`defaultFormat: "svg"`), иначе PNG. Для секций Integrations/Trust скачай каждый логотип.
- Декоративные эллипсы-glow на фоне — можно воспроизвести CSS (radial-gradient + blur), если это просто размытые круги.
- Чтобы скачать по временной ссылке: `curl -s -o <путь> "<url>"`.

## Как получить дизайн секции
Сначала загрузи схемы Figma-инструментов:
`ToolSearch` query: `select:mcp__claude_ai_Figma__get_design_context,mcp__claude_ai_Figma__get_screenshot,mcp__claude_ai_Figma__download_assets,mcp__claude_ai_Figma__get_metadata`
Затем:
- `get_design_context` (fileKey, nodeId) — референс-код + тексты + URL ассетов.
- `get_screenshot` (fileKey, nodeId, maxDimension 1400) — визуал; скачай через curl и посмотри Read'ом.
- `download_assets` (fileKey, nodeId) — растровые исходники.
